# BAB 4: PROTOKOL KEAMANAN & PRIVASI DATA

### 4.1 Arsitektur Autentikasi: JSON Web Token (JWT) Hardening
LocalMart mengadopsi standar industri **JSON Web Token (JWT)** sebagai pilar utama sistem autentikasi dan otorisasi. Penggunaan JWT pada LocalMart dirancang dengan prinsip *hardening* untuk memastikan setiap pertukaran data antara klien (Mobile/Web) dan server berjalan dalam koridor keamanan yang ketat. Kami menggunakan algoritma **HS256 (HMAC with SHA-256)** untuk melakukan *signing* pada setiap token yang diterbitkan. Keamanan kunci rahasia (*Secret Key*) dikelola secara eksklusif melalui **Environment Variables** (.env) yang diisolasi dari kode sumber, mencegah risiko kebocoran kunci melalui kontrol versi (Git).

Sistem autentikasi kami bersifat **Stateless (Sessionless)**. Keuntungan teknis dari arsitektur *stateless* ini adalah backend Go tidak perlu menyimpan status sesi pengguna di dalam basis data atau memori server (seperti Redis). Setiap token JWT membawa klaim identitas pengguna yang terenkripsi dan dapat divalidasi secara instan oleh server melalui pencocokan tanda tangan digital. Hal ini memberikan kemampuan skalabilitas yang luar biasa; backend LocalMart dapat menangani lonjakan pengguna secara masif (misalnya saat promosi daerah) tanpa terbebani oleh manajemen memori sesi, sekaligus menjaga performa tetap konsisten.

### 4.2 Network Security: CORS Policy & Header Protection
Keamanan jaringan pada level aplikasi diperketat melalui kebijakan **CORS (Cross-Origin Resource Sharing)** yang disiplin. Berbeda dengan konfigurasi umum yang seringkali membiarkan akses terbuka (*Wildcard*), LocalMart menerapkan kebijakan **Specific Origins**. Kami mengunci akses API hanya untuk domain atau alamat IP yang telah terverifikasi, yaitu Admin Panel resmi dan infrastruktur aplikasi Flutter. Semua permintaan (*request*) yang berasal dari asal (*origin*) di luar daftar putih tersebut akan ditolak secara otomatis oleh middleware backend, mencegah serangan bertipe *Cross-Site Request Forgery* (CSRF) dan penggunaan API yang tidak sah.

Selain kebijakan CORS, kami mengimplementasikan pertahanan melalui **Security Headers** pada setiap respon HTTP untuk memitigasi serangan di tingkat peramban:
*   **X-Frame-Options (DENY):** Mencegah aplikasi dimuat di dalam `iframe` pada situs asing, secara efektif melumpuhkan upaya serangan *Clickjacking*.
*   **X-Content-Type-Options (nosniff):** Memaksa peramban untuk mengikuti tipe MIME yang didefinisikan server, mencegah serangan *MIME Sniffing* yang sering digunakan untuk menyuntikkan skrip berbahaya melalui file gambar.
*   **Content-Security-Policy (CSP):** Membatasi sumber skrip dan aset yang boleh dijalankan, memberikan lapisan perlindungan tambahan terhadap *Cross-Site Scripting* (XSS).

### 4.3 Protokol Penghapusan Data & Self-Destruction Logic (Privacy-First)
LocalMart menjunjung tinggi hak privasi pengguna melalui implementasi protokol penghapusan data yang transparan dan tuntas. Kami memperkenalkan algoritma **Active Order Guard**; sebuah sistem proteksi yang akan melakukan validasi silang terhadap status transaksi sebelum mengizinkan penghapusan akun atau toko. Sistem akan menolak permintaan penghapusan jika masih ditemukan transaksi yang berstatus aktif (sedang dalam proses atau pengiriman). Hal ini krusial untuk melindungi hak konsumen dalam mendapatkan barangnya serta hak penjual untuk menyelesaikan kewajiban finansialnya.

Setelah validasi *Active Order Guard* terpenuhi, sistem akan menjalankan **Physical Asset Cleanup**. Saat data toko atau akun dihapus dari basis data, backend Go secara otomatis akan menelusuri jalur penyimpanan file fisik gambar produk dan logo di storage server. Melalui fungsi `os.Remove`, sistem akan menghapus aset-aset tersebut secara permanen. Langkah ini memastikan bahwa tidak ada residu data (*digital footprints*) yang tersisa di server kami, mencegah potensi kebocoran data (*data leakage*) di masa depan dan menjamin bahwa privasi pengguna benar-benar terlindungi secara fisik, bukan sekadar secara logis di level database.

### 4.4 Enkripsi Data & Prepared Statements
Integritas data di tingkat penyimpanan diperkuat dengan penggunaan **Prepared Statements** yang dikelola oleh GORM. Setiap query ke database diproses menggunakan parameter terpisah, yang secara otomatis mengisolasi input pengguna dari perintah SQL. Mekanisme ini memberikan proteksi 100% terhadap serangan **SQL Injection**, yang sering menjadi pintu masuk utama kebocoran data pada aplikasi web.

Untuk data yang dikategorikan sangat sensitif (seperti data pribadi yang diwajibkan oleh UU Pelindungan Data Pribadi), LocalMart menerapkan **Application-Level Encryption**. Data tersebut dienkrpisi sebelum disimpan ke dalam file `.db` SQLite menggunakan algoritma enkripsi standar industri. Dengan protokol ini, meskipun terjadi skenario terburuk di mana file basis data berhasil diakses secara ilegal oleh pihak luar, data sensitif di dalamnya tetap tidak terbaca tanpa kunci enkripsi yang tersimpan aman di server aplikasi. Pendekatan keamanan berlapis ini menjadikan LocalMart sebagai platform yang tangguh dan terpercaya bagi masyarakat Kabupaten Sumbawa Barat.
